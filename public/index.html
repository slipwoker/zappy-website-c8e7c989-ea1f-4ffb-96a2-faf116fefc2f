Perguntas Frequentes

Tudo o que você precisa saber sobre o Zaply para começar a automatizar suas mensagens.

O Zaply é gratuito?

Sim! O Zaply é um projeto open-source hospedado no GitHub sob a licença MIT. Você pode clonar o repositório, modificar o código e usá-lo gratuitamente em seus próprios servidores. Não há custos de licenciamento — apenas os custos operacionais da sua infraestrutura e da API do WhatsApp Business, caso utilize.

Preciso de uma conta WhatsApp Business API?

Sim, para enviar mensagens programáticas em produção é necessário ter acesso à WhatsApp Business API. O Zaply atua como uma camada de automação que se conecta a essa API. Você pode utilizar provedores oficiais como a Meta ou soluções de terceiros compatíveis com a API. Para testes locais, é possível simular o comportamento com mocks.

Consigo enviar mensagens em massa?

Com certeza. O Zaply foi projetado justamente para facilitar o envio de mensagens em massa de forma programática. Através de sua API interna, você pode disparar campanhas para listas de contatos com agendamento, personalização de conteúdo e controle de intervalos entre envios, respeitando os limites da plataforma para evitar bloqueios.

Existe limite de taxa de envio?

O Zaply em si não impõe um rate limit artificial, mas a entrega está sujeita às políticas de uso da WhatsApp Business API. Dependendo do seu plano e provedor, há limites de mensagens por segundo e por dia. O Zaply inclui mecanismos de fila e retry com backoff exponencial para lidar com essas restrições de forma inteligente, garantindo a máxima entregabilidade.

Como faço o deploy do Zaply no meu servidor?

O deploy é simples e direto, feito sob medida para desenvolvedores. Basta clonar o repositório do GitHub, configurar as variáveis de ambiente com suas credenciais de API e executar via Docker Compose ou diretamente com Node.js. O README do projeto contém um guia passo a passo, incluindo exemplos para deploy em VPS, AWS, Railway ou qualquer plataforma que suporte containers.

Existe suporte oficial?

Por ser um projeto open-source, o suporte principal é conduzido pela comunidade através das Issues e Discussions no GitHub. No entanto, mantenedores e colaboradores frequentes respondem ativamente a dúvidas e revisam pull requests. Para empresas que necessitam de suporte prioritário, SLA garantido ou implementações personalizadas, oferecemos planos de consultoria — entre em contato conosco para saber mais.
